## 引擎认为玩家有两个状态

- 存活，即 `player:Alive() == true`，第一人称视角，绘制 view model，玩家可以自由战斗
- 死亡，即 `player:Alive() == false`，第三人称视角，不绘制 view model，生成一个布娃娃作为玩家的尸体

## 引擎的状态转移

存活时，当玩家受到伤害后，如果 `player:Health() <= 0`，则进入死亡
死亡时，如果玩家按下鼠标左键，则进入存活

## 本 MOD 将玩家状态扩展为按血量划分的五个主状态，其中挣扎内部有按时间划分的两个子状态

- 战斗
- 挣扎
  - 死亡动画
  - 爬行
- 抽搐
- 死亡

引入概念

- 战斗血量 `player.context.combatHealth = player:GetMaxHealth() -- 假设返回 100`
- 挣扎血量 `player.context.strugglingHealth -- 为运行时常量，假设配置为 50`
- 抽搐血量 `player.context.writhingHealth -- 为运行时常量，假设配置为 25`
- 额外血量 = 爬行血量 + 挣扎血量 `player.context.additionalHealth -- 为运行时变量，此时计算为 75`
- 显示血量，即引擎管理和认为的血量，此时为 100

## 本 MOD 的状态转移

每当玩家受到伤害前，即 `ScalePlayerDamage` 钩子触发时，本 MOD 先通过

```lua
local damage = dmginfo:GetDamage()
local logicHealthBefore = player:Health() + player.context.additionalHealth
local logicHealthAfter = logicHealthBefore - damage

if logicHealthAfter <= 0 then return end -- 击穿了所有额外血量，直接返回让引擎处理玩家死亡
if logicHealthAfter <= player.context.writhingHealth then
    local keptHealth = 0 -- 将所有运行时额外血量在伤害应用前转移过来，恰能阻止死亡
    player:SetMaxHealth(player:GetMaxHealth() +  player.context.additionalHealth - keptHealth) -- 伤害结算后恢复为 player:GetMaxHealth() https://wiki.facepunch.com/gmod/GM:PostEntityTakeDamage
    player:SetHealth(player:Health() +  player.context.additionalHealth - keptHealth)
    player.context.additionalHealth = keptHealth
    player.context.state = "writhing"
    return
end
if logicHealthAfter <= player.context.writhingHealth + player.context.strugglingHealth then
    local keptHealth = player.context.writhingHealth
    player:SetMaxHealth(player:GetMaxHealth() +  player.context.additionalHealth - keptHealth) -- 伤害结算后恢复为 player:GetMaxHealth()
    player:SetHealth(player:Health() +  player.context.additionalHealth - keptHealth)
    player.context.additionalHealth = keptHealth
    player.context.state = "struggling"
    return
end
if logicHealthAfter > player.context.writhingHealth + player.context.strugglingHealth then
    player.context.state = "combat"
    return
end
```
